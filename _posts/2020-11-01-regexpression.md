### 什么是正则表达式？    
***正则表达式*** 又叫做 ***规则表达式***。是使用一组经过特殊定义的表达式，来检索、替换那些符合某个规则的文本。    

例如：可以使用 '\d' 匹配数字，下表是一些常见的正则表达式匹配规则

|  正则表达式   | 匹配规则  |
|  ----  | ----  |
| \d  | 匹配 一个 0~9 之间的数字 |
| [0-9]  | 匹配 一个 0~9 之间的数字，和\d功能一样 |
| \D  |  匹配 一个 非数字    |
| [^0-9]|匹配 一个非数字 和 \D 功能一样|


### 正则表达式的符号定义    
巧妇难为无米之炊，正则表达式也有一些最基本的元素。下表是一些正则表达式的符号定义。    
#### 基本符号    

| 符号 |符号含义|
| ----- | ---- |
| [] |接受的字符列表，[abc] 表示a or b or c|
| [^] | 不接受的字符列表，匹配除  xxxx 的，例如 [^abc]，除a、b、c之外的|
| \ |转义字符，例如\\\，这里就是符号 '\' |    
| \| | 或者，ac\|bd表示ac或者bd |
| - | 连字符 \-, A-Z 表示A-Z之间的字符 |    

#### 限定符    

| 符号 |符号含义|
| ----- | ---- |
| * | 指定字符可以重复 0 ~ n次 |
| + | 指定字符可以重复 1 ~ n次 |
| ? | 指定字符可以重复 0 ~ 1次 |
| {n} | 只能输入n个字符 |
| {n, }| 至少n个匹配|
| {n,m} | 指定 n ~ m个字符 |
| ^ | 指定起始字符 |
| $ | 指定结束字符 |      



#### 匹配字符集    

| 符号 |符号含义|
| ----- | ---- |
| \d  | 匹配 一个 0~9 之间的数字 |
| \D  |  匹配 一个 非数字    |
| \w | 匹配单个数字、大小写字母、相当于[0-9a-zA-Z] |
| \W | 匹配单个非数字、大小写字母，相当于[^0-9a-zA-Z] |     

### 一些常用的正则表达式    
在日常开发工作中，我们经常会遇到匹配常用号码、验证一些输入是否满足一定的规则的需求。下面是一些常用的正则表达式，需要的小伙伴可以直接拿去用。    

| 正则表达式 | 含义 |  
| ---- | ---- |
|^\d+$| 非负整数|
| ^-{1}\d+$ | 负数 |
| ^-?\d+$ |  整数 |
| ^[A-Za-z]+$ | 英文字符串 |
| ^[A-Za-z0-9_]+$ | 字母、数字、下划线组合 |
|^[0-9]{6}| 匹配六位数字(银行卡密码) |
|^\w+([-+.]\w+)*@\w+([-.]\w+)*\.\w+([-.]\w+)*$| Email地址 |
|^\d{4}-\d{1,2}-\d{1,2}| 日期格式 yyyy-MM-dd这种|

### Java中使用正则表达式    

```java
class RegularExpression {
	public static void main(String[] args) {
		String reg = "^[a-z]{3}$";
		System.out.println("abcd".matches(reg));
		System.out.println("def".matches(reg));
	}
}
```    
**String** 类有个成员方法 **matches(String reges)**，返回值为boolean，用于告诉这个字符串是否匹配给定的正则表达式。在上面的例子中，我们给出的正则表达式为^[a-z]{3}，表示匹配3位小写英文字母。显然***def***是匹配的，返回true。但***abcd***不满足条件返回false;    

在 **java.util.regex**包下有两个用于正则表达式的类，一个是**Matcher**类，另一个是**Pattern**类。Java手册中给了这两个类的典型用法，代码如下： 

```java
import java.util.regex.*;

class RegularExpression {
	public static void main(String[] args) {
		Pattern p = Pattern.compile("[a-z]{3}");
		Matcher m = p.matcher("abc");
		System.out.println(m.matches());
	}
}

//输出结果
true
```        

* 正则表达式背后的执行原理，涉及到编译原理中的自动机等知识，这里不展开细说。   
* **Pattern**可以理解为一个模式，字符串需要与某种模式进行匹配。
* 创建**Pattern**对象调用的是一个静态方法**Pattern.compile**。这里就将一个字符串的正则表达式编译为了一个常量，可以安全供给多个线程并发使用。
* **Matcher**是字符串和某个模式匹配后可能会产生的多个结果。最后当我们调用**m.matches()**时，就会返回完整字符串与模式匹配的结果。
